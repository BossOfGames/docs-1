VSE API Docs

These are the API Docs

Pirep​

Id​

string: The phpVMS PIREP ID

FlightId​

string: The ID of the flight from phpVMS

Example​
pirep.

SimType​

string: The simulator type they're using

  • Prepar3d
  • X-Plane
  • Microsoft Flight Simulator
  • FSX
  • FS9

Ident​

string

FlightNumber​

string: The flight number

RouteLeg​

string: The route leg

RouteCode​

string: The route code

FlightType​

string

PlannedDistance​

number: Distance planned Might not always be set or available

ActualDistance​

number: The distance when the flightplan is measured Might not always be set or available

DistanceToGo​

number: How far they ahve already travelled Might not always be set or available

EstimatedTravelTime​

number: The amount of time estimated that this should take. In minutes Might not always be set or available

Airline​

Airline: The airline the pilot is a part of

Aircraft​

Aircraft: The aircraft the pilot is flying

Flight​

Flight: The flight this PIREP originated from, if it was from a bid, or loaded on the screen

Features​

Features: The startup variables/state when this PIREP started

FlightPlan​

FlightPlan

IsInActiveState​

Boolean: Is this flight in an active state?

State​

string: The PIREP states - these match the phase

  • Not Running
  • Initialized
  • Boarding
  • Pushback
  • Taxi Out
  • Takeoff
  • Enroute
  • Approach
  • Final
  • Landed
  • Taxi In
  • On Block
  • Arrived
  • Cancelled
  • Filed
  • Paused

Phase​

string

  • INI - Initialized
  • BST - Boarding started
  • PBT - Pushback started
  • TXI - Taxi
  • TOF - Takeoff
  • ENR - Enroute
  • APR - Approach
  • FIN - On Final
  • LAN - Landing
  • TXI - Taxi
  • ONB - On-block
  • ARR - Arrived
  • CNL - Cancelled
  • FIL - Filed
  • PSD - Paused

DepartureAirport​

Airport: The airport they departed from Might be null if it wasn't detected

DepartureGate​

Gate: The departure gate, null if they aren't at one. Might be null if it wasn't detected

Example​
console.log(pirep.DepartureGate)

DepartureRunway​

Runway: The runway they took off from Might be null if it wasn't detected

CrossingRunway​

Runway: This will have a value, with the runway information, if they're currently on a runway Might be null if it wasn't detected

ArrivalAirport​

Airport: The arrival airport, null if they haven't landed yet

ArrivalRunway​

Runway: The runway that they landed on. Might be null if it wasn't detected

ArrivalGate​

Gate: The arrival gate, null if they aren't at one. Might be null if it wasn't detected

Route​

string

number: Unix timestamp, the time in the sim

PlannedFlightTime​

System.Nullable{number}: Gets or sets the planned flight time. The planned flight time.

IsResuming​

Boolean: If this PIREP is being resumed

StartedWithBrakes​

Boolean: Did they start the flight with the brakes on? Use this to determine the criterea on moving out of the Boarding phase

DirectTakeoff​

Boolean: Is this a direct takeoff? That means when in "boarding", the OnGround flag is immediately detected as true

Runway: Save the runway they're on. This gets set if they're taxiing too, so we can keep track of if we're crossing a runway or something If the pilot is on the runway
